# Agent operating contract

## Canon

`ASOLARIA OS-ON-METAL = frozen USB substrate + HyperBEHCS active thinking OS`.
The frozen weights bank is never mounted or formatted. HBP/HBI/SHA receipts,
PID-crypto identity, MTP thoughts, and Atlas tiering remain distinct types.

## Non-negotiable boundaries

- Never commit raw Wikipedia, GGUF/model weights, matrices, keys, credentials,
  absolute seat paths, drive inventories, or private recall contents.
- Acer, Liris, and Relic exchange only derived projections, executable public
  glyph functions, catalog commitments/addresses, and Omega bindings.
- An address or Omega commitment is not missing entropy and cannot be described
  as a substitute for decoder/model bytes.
- The 3,200-byte Quant8 packet is the original route-tail layer, not proof that
  Wikipedia is reconstructible from 3.1 KB.
- Mathematical reflection, lens, gravity, quantum, plasma, universe, and
  flashlight terms are simulated-computation language unless physical evidence
  is independently supplied.
- Preserve operator numbers as separate axes; never use one to overwrite
  another.

## Evidence labels

- `MEASURED`: reproduced from executable code or byte evidence.
- `CANON`: ratified project law or signed receipt.
- `OPERATOR_SPECIFIED`: exact operator target not yet independently measured.
- `DESIGN`: implementation contract.
- `UNVERIFIED`: no adequate receipt.

## Tournament law

Each seat branch must:

1. use both reflection fields A/B;
2. consume commitments or public fixtures only;
3. keep predictions causal;
4. account for all transmitted and regenerated state;
5. produce an exact replay hash before claiming losslessness;
6. write a receipt without local paths or corpus bytes.

No branch may claim a Hutter result from gain estimates, compression ratios that
omit the decoder, or a route/commitment slice alone.
